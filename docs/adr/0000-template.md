# NNNN. Título corto de la decisión

- **Estado**: Aceptada | Superseded by [ADR-NNNN](./NNNN-titulo.md) | Obsoleta
- **Fecha**: AAAA-MM-DD

## Contexto

Qué problema o restricción forzó esta decisión. Solo el contexto necesario para entenderla, no una recapitulación de toda la fase.

## Decisión

Qué se decidió, en una o dos frases directas.

## Alternativas consideradas

- **Opción A**: por qué se descartó.
- **Opción B**: por qué se descartó.

## Consecuencias

Trade-offs aceptados a cambio de esta decisión — lo que se gana y lo que se renuncia a cambio, no solo lo positivo.

---

Un ADR es atómico e inmutable: registra una decisión, no se edita después. Si la decisión cambia, se crea un ADR nuevo que la sustituye (`Superseded by`) y se actualiza el `Estado` de este a `Superseded by [ADR-NNNN]`, dejando el razonamiento original intacto y consultable. Numeración secuencial (`0001-...`, `0002-...`), un archivo por decisión.
